# AEO-GEO Project Lab

This project helps you learn and execute:
- AEO (Answer Engine Optimization)
- GEO (Generative Engine Optimization)

Goal:
- Build and test real-world content optimization workflows for both AEO and GEO.
- Compare outcomes with measurable metrics.

## Project Structure

- docs/: guides and execution steps
- experiments/aeo/: AEO experiment notes and outputs
- experiments/geo/: GEO experiment notes and outputs
- experiments/comparison/: side-by-side analysis
- real-world-tests/: live test runs and screenshots
- templates/: reusable templates
- data/raw/: source keyword/query/input data
- data/processed/: cleaned and enriched experiment data

## Quick Start

1. Read docs/IMPLEMENTATION_STEPS.md fully.
2. Pick one niche topic for testing (example: health insurance claim FAQs).
3. Prepare baseline content and save in experiments/aeo and experiments/geo.
4. Run Week 1 setup and Week 2 optimization tasks.
5. Use templates/experiment-tracker.csv to log every test.
6. Run comparison tests in experiments/comparison.
7. Summarize results in docs/WEEKLY_REVIEW.md.

## What To Measure

Core KPIs:
- Visibility in answer-style results (AEO)
- Citation or mention frequency in AI responses (GEO)
- Click-through rate from search (if available)
- Accuracy and completeness score of answers
- Conversion proxy (lead form click, demo click, or contact action)

## Suggested Real-World Channels

- Google Search snippets and People Also Ask
- ChatGPT, Gemini, Perplexity, Copilot style prompts
- Niche forums and Q&A platforms
- YouTube transcript optimization for Q&A queries

## Output of This Lab

By the end you should have:
- AEO-ready content blocks and FAQ structures
- GEO-ready entity-rich, citation-friendly long-form content
- A data-backed comparison report of AEO vs GEO performance
- A repeatable testing workflow
